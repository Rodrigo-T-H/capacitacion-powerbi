Sesión 1: Fundamentos de Power BI

1. Introducción a Power BI

Microsoft Power BI es una suite de herramientas de inteligencia de negocios que permite conectar, transformar, modelar y visualizar datos para crear informes interactivos y cuadros de mando.

¿Por qué Power BI en Retail?

  • • Toma de decisiones basada en datos: Transforma datos brutos en información valiosa
  • • Monitoreo de métricas clave: Ventas, inventarios y rendimiento de productos
  • • Identificación de tendencias: Patrones y oportunidades del consumidor
  • • Optimización de estrategias: Mejora campañas con insights precisos

2. Preparación de Datos

La calidad de tus datos es crucial para un análisis preciso. Esta sección te guiará a través de los pasos para conectar y limpiar tus datos.

Limpieza Previa a la Integración

Antes de cargar datos en Power BI, es vital realizar una limpieza inicial para asegurar la calidad y consistencia.

Buscar y corregir valores faltantes, inconsistencias o errores en los datos que puedan distorsionar el análisis. Incluye revisar formatos de fecha, valores numéricos fuera de rango y campos obligatorios vacíos.

Asegurar que los nombres de columnas y los formatos de datos (fechas, números, texto) sean consistentes en todas las fuentes. Esto incluye unificar nomenclaturas y estructuras de datos.

Verificar que los datos cumplan con reglas de negocio o rangos esperados para garantizar su integridad. Por ejemplo, verificar que las fechas estén en rangos válidos y que los precios sean positivos.

3. Modelado de Datos para Retail

El modelado de datos es el proceso de definir las relaciones entre tus tablas para que Power BI pueda entender cómo se conectan y realizar cálculos precisos.

Relaciones entre Tablas

Las relaciones permiten que los filtros aplicados en una tabla afecten a otras tablas relacionadas, creando un modelo de datos coherente.

Creación de un Modelo Simple

Un modelo básico para retail incluye estas tres tablas principales, siguiendo un esquema de estrella:

📊 Tabla de Ventas (Hechos)

Contiene los detalles de cada transacción: ID de transacción, fecha, monto, cantidad, ID de producto, etc.

🏷️ Tabla de Productos (Dimensión)

Información sobre cada producto: ID de producto, nombre, categoría, precio, marca, etc.

📅 Tabla de Fechas (Dimensión)

Tabla de calendario con todas las fechas relevantes: año, mes, día, trimestre, día de la semana, etc.

5. DAX: Fórmulas y Medidas

DAX (Data Analysis Expressions) es el lenguaje de fórmulas de Power BI que permite crear medidas que realizan cálculos dinámicos sobre tus datos.

¿Qué es DAX?

DAX es una colección de funciones, operadores y constantes que se pueden usar para calcular y devolver valores. Es la clave para obtener insights profundos de tus datos.

  • ✓ Cálculos dinámicos
  • ✓ Medidas contextuales
  • ✓ Análisis de tiempo
  • ✓ Comparaciones avanzadas
Ejemplo de Medida DAX:
Total Ventas = 
SUM(Ventas[Monto])

Esta medida suma todos los valores de la columna Monto de la tabla Ventas.

Medidas Básicas para Retail

Calcula la suma total de los montos de ventas.

Total de Ventas = SUM(Ventas[Monto])
📖 Función SUM en DAX →

Cuenta el número de transacciones únicas.

Conteo Transacciones = COUNT(Ventas[ID_Transacción])
📖 Función COUNT en DAX →

Calcula el valor promedio de cada transacción.

Ticket Promedio = DIVIDE([Total de Ventas], [Conteo Transacciones])
📖 Función DIVIDE en DAX →

Calcula el porcentaje de margen de ganancia.

% Margen = DIVIDE([Ganancia Total], [Total de Ventas])

Esta medida requiere que tengas una columna o medida de Ganancia Total.

Funciones Avanzadas

DAX ofrece funciones potentes para manipular el contexto de filtro y realizar cálculos complejos.

Evalúa una expresión en un contexto de filtro modificado.

Ventas Electrónica = 
CALCULATE(
    [Total de Ventas],
    Productos[Categoría] = "Electrónica"
)
📖 Función CALCULATE →

Compara períodos y analiza tendencias temporales.

Ventas Año Anterior = 
CALCULATE(
    [Total de Ventas],
    SAMEPERIODLASTYEAR(Fechas[Fecha])
)
📖 DAX Guide - Funciones de Tiempo →

Las variables mejoran la legibilidad y el rendimiento al evitar recálculos.

Medida con VAR =
VAR TotalVentas = [Total de Ventas]
VAR TotalTransacciones = [Conteo Transacciones]
RETURN
    DIVIDE(TotalVentas, TotalTransacciones)
